# How to Access Redux, Context API, and useReducer in Your App\n\n## 1. USING THE useAuth() HOOK (Context API)\n\nThis is the **easiest** way to access auth state in any component.\n\n### Example in HomeScreen.js\n```javascript\nimport { useAuth } from '../context/AuthContext';\n\nexport default function HomeScreen() {\n  // Access auth state and methods\n  const { user, isAuthenticated, logout, isLoading, error } = useAuth();\n  \n  if (!isAuthenticated) {\n    return <Text>Please log in first</Text>;\n  }\n  \n  return (\n    <View>\n      <Text>Welcome, {user.displayName}!</Text>\n      <Text>Email: {user.email}</Text>\n      <TouchableOpacity onPress={logout}>\n        <Text>Logout</Text>\n      </TouchableOpacity>\n    </View>\n  );\n}\n```\n\n### Available from useAuth()\n```javascript\nconst {\n  // STATE\n  user,              // { id, email, displayName, createdAt } or null\n  token,             // JWT string or null\n  isAuthenticated,   // true/false\n  isLoading,         // true/false (during login/signup)\n  isHydrated,        // true/false (app startup complete)\n  error,             // error message or null\n  successMessage,    // success message or null\n  \n  // METHODS\n  login,             // async (email, password) => Promise\n  signUp,            // async (email, password, displayName) => Promise\n  logout,            // async () => Promise\n  clearError,        // () => void - clear error message\n  clearSuccess,      // () => void - clear success message\n} = useAuth();\n```\n\n---\n\n## 2. USING REDUX SELECTORS (Direct Redux Access)\n\nIf you want to access Redux state directly (less common, mostly use useAuth):\n\n### Example in ProfileScreen.js\n```javascript\nimport { useSelector } from 'react-redux';\n\nexport default function ProfileScreen() {\n  // Select from Redux store\n  const auth = useSelector(state => state.auth);\n  const user = useSelector(state => state.auth.user);\n  const isLoading = useSelector(state => state.auth.isLoading);\n  \n  return (\n    <View>\n      <Text>User: {user?.displayName}</Text>\n      <Text>Loading: {isLoading ? 'Yes' : 'No'}</Text>\n      <Text>Authenticated: {auth.isAuthenticated ? 'Yes' : 'No'}</Text>\n    </View>\n  );\n}\n```\n\n### Or Select Multiple at Once\n```javascript\nconst { user, isLoading, isAuthenticated } = useSelector(state => ({\n  user: state.auth.user,\n  isLoading: state.auth.isLoading,\n  isAuthenticated: state.auth.isAuthenticated,\n}));\n```\n\n⚠️ **Note:** Most of the time, use `useAuth()` instead of `useSelector()` - it's simpler!\n\n---\n\n## 3. DISPATCHING REDUX ACTIONS (Advanced)\n\nIf you want to dispatch Redux actions directly:\n\n### Example\n```javascript\nimport { useDispatch } from 'react-redux';\nimport { loginAsync, logout } from '../store/slices/authSlice';\n\nexport default function Component() {\n  const dispatch = useDispatch();\n  \n  const handleLogin = async () => {\n    // Dispatch async thunk\n    const result = await dispatch(loginAsync({\n      email: 'user@email.com',\n      password: 'password123'\n    }));\n    \n    // Check result\n    if (result.meta.requestStatus === 'fulfilled') {\n      console.log('Login success:', result.payload);\n    } else {\n      console.log('Login failed:', result.payload);\n    }\n  };\n  \n  const handleLogout = () => {\n    // Dispatch regular action\n    dispatch(logout());\n  };\n  \n  return (\n    <>\n      <Button onPress={handleLogin} title=\"Login\" />\n      <Button onPress={handleLogout} title=\"Logout\" />\n    </>\n  );\n}\n```\n\n⚠️ **Note:** Use `useAuth()` instead! It handles all this for you.\n\n---\n\n## 4. USING useReducer FOR FORM STATE (Local Component State)\n\nUsed in LoginScreen for managing form inputs locally.\n\n### How It Works\n```javascript\nimport { useReducer } from 'react';\n\n// Step 1: Define reducer function\nconst formReducer = (state, action) => {\n  switch (action.type) {\n    case 'SET_EMAIL':\n      return { ...state, email: action.payload, emailError: '' };\n    case 'SET_PASSWORD':\n      return { ...state, password: action.payload, passwordError: '' };\n    case 'SET_ERRORS':\n      return { ...state, ...action.payload };\n    case 'RESET':\n      return { email: '', password: '', emailError: '', passwordError: '' };\n    default:\n      return state;\n  }\n};\n\n// Step 2: Define initial state\nconst initialState = {\n  email: '',\n  password: '',\n  emailError: '',\n  passwordError: '',\n};\n\n// Step 3: Use in component\nexport default function FormComponent() {\n  const [form, dispatch] = useReducer(formReducer, initialState);\n  \n  return (\n    <>\n      <TextInput\n        value={form.email}\n        onChangeText={(text) => \n          dispatch({ type: 'SET_EMAIL', payload: text })\n        }\n      />\n      {form.emailError && <Text>{form.emailError}</Text>}\n      \n      <TouchableOpacity \n        onPress={() => dispatch({ type: 'RESET' })}\n      >\n        <Text>Clear Form</Text>\n      </TouchableOpacity>\n    </>\n  );\n}\n```\n\n---\n\n## 5. QUICK REFERENCE - WHERE TO USE WHAT\n\n### useAuth() - USE THIS MOST OF THE TIME ✅\n**Where:** Any component that needs auth state  \n**Why:** Simple, clean, no Redux boilerplate  \n**Example:** `const { user, login, logout } = useAuth();`\n\n### useReducer - FOR FORM STATE ✅\n**Where:** LoginScreen, SignUpScreen, edit forms  \n**Why:** Manage complex form state locally  \n**Example:** `const [form, dispatch] = useReducer(formReducer, initialState);`\n\n### useSelector - ONLY IF NEEDED ⚠️\n**Where:** Rarely needed, advanced cases  \n**Why:** Direct Redux access  \n**Example:** `const user = useSelector(state => state.auth.user);`\n\n### useDispatch - ONLY IF NEEDED ⚠️\n**Where:** Rarely needed, advanced cases  \n**Why:** Direct Redux action dispatch  \n**Example:** `const dispatch = useDispatch();`\n\n---\n\n## 7. HOW DATA FLOWS\n\n```\nUSER TYPES EMAIL\n        ↓\nLOCAL FORM REDUCER\n(useReducer manages email input)\n        ↓\nVALIDATION\n(JavaScript validation in component)\n        ↓\nCALL useAuth() METHOD\n(login(email, password))\n        ↓\nCONTEXT API DISPATCHES TO REDUX\n(loginAsync thunk)\n        ↓\nREDUX STATE UPDATES\n(auth.isLoading = true, auth.user = {...})\n        ↓\nCOMPONENT RE-RENDERS\n(useAuth() subscribed to state change)\n        ↓\nASYNCSTORAGE PERSISTS\n(login persisted to device)\n        ↓\nNAVIGATION UPDATES\n(RootNavigator sees isAuthenticated = true)\n```\n\n---\n\n## 8. TESTING IN YOUR APP\n\n### Test 1: Access user in any component\n```javascript\nimport { useAuth } from '../context/AuthContext';\n\n// In any component:\nconst { user } = useAuth();\nconsole.log(user.displayName); // \"Vedant\" or whatever they signed up as\n```\n\n### Test 2: Try logout from ProfileScreen\n```javascript\nconst { logout } = useAuth();\n\n<TouchableOpacity onPress={() => logout()}>\n  <Text>Logout</Text>\n</TouchableOpacity>\n// Walks back to LoginScreen automatically\n```\n\n### Test 3: Check isLoading state\n```javascript\nconst { isLoading, login } = useAuth();\n\n<TouchableOpacity onPress={() => login(...)} disabled={isLoading}>\n  <Text>{isLoading ? 'Loading...' : 'Login'}</Text>\n</TouchableOpacity>\n```\n\n---\n\n## KEY CONCEPTS EXPLAINED\n\n### Context API (AuthContext.js)\n- ✅ Shares auth state across entire app without prop drilling\n- ✅ useAuth() hook makes it easy to access\n- ✅ Combines Redux state + AsyncStorage + navigation\n- ✅ Use this in 95% of cases\n\n### Redux (authSlice.js)\n- ✅ Centralized state management\n- ✅ Predictable state updates\n- ✅ Async thunks handle API calls\n- ✅ Used internally by Context API\n- ✅ Direct access optional (rarely needed)\n\n### useReducer (in components)\n- ✅ Local form state management\n- ✅ Handles complex form logic\n- ✅ Not shared across components\n- ✅ Use for login form, edit forms, etc.\n\n### Integration\n- Redux + Context API = Global auth state ✅\n- useReducer + Context = Form-level state ✅\n- Together = Perfect app architecture 🚀\n"
